## 项目概述

本项目是一个基于**遗传算法**的卡车-无人机协同配送路径优化系统，用于解决物流配送中的最优路径规划问题。

### 核心功能

- ✅ **卡车-无人机协同配送**：卡车仅支援无人机，无人机负责最终配送
- ✅ **单仓库支持**：支持单个配送仓库
- ✅ **时间优化**：专注最小化总配送时间
- ✅ **发射点自动优化**：K-means++聚类初始化 + GA优化
- ✅ **客户分块分配**：按聚类分配客户到发射点
- ✅ **时间窗约束**：支持客户配送时间窗限制
- ✅ **可视化界面**：Java Swing GUI界面
- ✅ **收敛曲线**：时间收敛曲线，坐标轴自动调整

## 配送模型说明

### 配送流程

```
卡车出发 → 行驶到发射点 → 无人机起飞 → 服务客户 → 无人机返回 → 卡车继续行驶 → ... → 返回仓库
```

### 核心特点

1. **卡车仅作母舰**：卡车不直接给客户送货，只负责携带无人机到发射点
2. **无人机负责配送**：无人机从发射点出发，给分配的客户服务后必须返回卡车
3. **发射点自动优化**：使用K-means++聚类初始化，支持发射点位置优化

## 染色体编码设计

### 三段式编码格式

```
[客户分块编码][发射点分配与路径编码][发射点坐标编码]

示例：[1, 1, 2, 2, 1, 3 | 1, 2, 3, 0 | 45.5, 60.2, 80.3, 55.8, 30.0, 70.0]
```

| 段 | 长度 | 说明 |
|----|------|------|
| 客户分块编码 | N（客户数） | 每个客户所属发射点编号（1~n） |
| 发射点分配与路径编码 | K×M_max向上取整 | 每辆卡车的发射点分配和访问顺序 |
| 发射点坐标编码 | 2×n | 每个发射点的(x, y)坐标 |

**参数说明**：
- n：发射点数量（使用肘部法则自动确定）
- K：卡车数量
- M_max = ceil(n/K)：每辆卡车最大分配发射点数

### K-means++聚类初始化

发射点数量通过肘部法则自动确定，约束条件：
- n ≥ K（发射点数 ≥ 卡车数）
- n < N/U（发射点数 < 客户数/每卡车无人机数）向上取整

初始化流程：
1. 使用肘部法则确定最优发射点数n
2. 若n不符合约束条件n ≥ K和n < N/U，则就近取n值为K（卡车数）或者N/U（客户数/每卡车无人机数）向上取整
3. 对客户位置进行K-means++聚类，分成n个聚类
4. 每个聚类的中心作为发射点初始位置
5. 每个聚类中的客户分配给对应的发射点

### 解码流程

```
步骤1: 解析客户分块 → 每个发射点服务的客户列表
步骤2: 解析发射点分配 → 每辆卡车的发射点访问顺序
步骤3: 解析发射点坐标 → 发射点位置
步骤4: 生成配送方案
       - 卡车按顺序访问发射点
       - 从发射点释放无人机
       - 无人机服务客户后返回
```

## 时间优化

### 优化目标

专注**最小化总配送时间**

### 适应度函数

```java
// 有效时间 = 总时间 + 惩罚时间（用于GA进化选择）
effectiveTime = totalTime + extraTime + distancePenaltyTime

// 适应度
fitness = 1.0 / (1.0 + effectiveTime)

// 最终输出实际配送时间（不含惩罚）
actualTime = totalTime
```

### 时间计算公式
1 卡车时间计算
卡车总时间 = 卡车行驶时间 + 等待时间

其中：
- 卡车行驶时间 = 总行驶距离 / 卡车速度 × 60
- 总行驶距离 = 仓库到第一个发射点距离 + 发射点间距离 + 最后一个发射点到仓库距离
- 等待时间 = Σ(每个发射点的无人机配送时间)

2 无人机时间计算

每个发射点的无人机配送时间 = max(各无人机在该发射点的配送时间)

无人机配送时间 = 飞行时间 + 服务时间 + 充电时间

其中：
- 飞行时间 = 2 × 发射点到客户距离 / 无人机速度 × 60  // 往返时间
- 服务时间 = 客户服务时间
- 充电时间 = 充电次数 × 单次充电时间
- 充电次数 = ceil(总飞行距离 / 无人机最大航程) - 1

3 总配送时间计算

总配送时间 = max(各卡车的总时间)

其中：
- 卡车总时间 = 卡车行驶时间 + 各发射点无人机配送时间之和



### 惩罚机制

```
有效时间 = 总时间 + 时间惩罚 + 距离惩罚

时间惩罚（分钟）:
  - 早到惩罚：min(到达时间 - 开始时间, 30) × timeWindowEarly
  - 晚到惩罚：min(到达时间 - 结束时间, 60) × timeWindowLate

无人机惩罚:
  - 续航超限：超出续航部分 × droneRangePenalty
  - 载重超限：超出载重部分 × dronePayloadPenalty

距离惩罚（分钟）:
  - 发射点到客户单程总距离 × distancePenalty
```

**注意**：所有惩罚最终转换为时间单位（分钟），便于统一比较。

### 输出说明

- **GA进化过程中**：使用有效时间（包含惩罚）进行选择
- **最终输出结果**：使用实际配送时间（不含惩罚）

## 遗传算法流程

```
1. 初始化种群（1个基础 + 90%扰动 + 10%随机）
2. 评估适应度（基于有效时间）
3. 选择（锦标赛选择）
4. 交叉（段交叉 + BLX-α坐标交叉）
5. 变异（交换变异 + 高斯变异）
6. 精英保留
7. 重复2-6直到收敛
```

### 关键算子

| 算子 | 类型 | 说明 |
|-----|------|------|
| 选择 | 锦标赛 | 规模=3，从种群中随机选3个最优 |
| 交叉 | 段交叉 + BLX-α | 完整复制一段，另一段BLX-α坐标交叉 |
| 变异 | 三段式变异 | 段1交换/重置，段2卡车交换，段3高斯变异 |

### 三段式变异详解

#### 段1：客户分块变异
- **70%概率**：交换变异，随机交换两个客户所属的发射点
- **30%概率**：随机重置，将随机一个客户变异到有效发射点编号

#### 段2：发射点分配变异（卡车间交换）
- 随机选择两辆不同的卡车
- 从第一辆卡车随机选一个发射点 l1
- 从第二辆卡车随机选一个发射点 l2
- 交换归属：l1 → 卡车2, l2 → 卡车1
- 目的：增强搜索能力，对适应度有实际影响

#### 段3：发射点坐标变异
- 自适应高斯变异：步长随迭代进行逐渐减小
- 范围：`Δ = baseRange × (1 - progress × 0.5)`，最小30%

## 更新日志

### v4.2 (2026-03)

**第二段变异增强**

- 修改 `GAOperators.java` 中第二段变异逻辑
- 原逻辑：只打乱卡车内部访问顺序（对适应度无影响）
- 新逻辑：不同卡车间交换发射点归属
- 增强算法搜索能力，对适应度有实际影响

### v4.1 (2026-03)

**统一解码器重构**

- 新增 `ChromosomeDecoder.java`，统一染色体解码逻辑
- 解码流程严格遵循 README 规范的四步骤
- 发射点归属采用轮询分配规则：发射点 j → 卡车 `((j-1) % K)`
- 修复遗传算子中段1和段2不一致导致的路线图显示问题
- `FitnessFunction` 和 `GARunner` 使用统一解码器
- 代码清理：移除未使用的遗传算子和辅助类

### v4.0 (2026-03)

**三段式染色体编码**

- 使用K-means++聚类进行客户分块
- 肘部法则自动确定发射点数量
- 三段式编码：[客户分块][发射点分配][发射点坐标]
- 移除动态发射点功能

### v3.2 (2026-02)

**单目标时间优化**

- 移除成本相关代码，专注时间最优
- 简化适应度函数：`fitness = 1 / (1 + effectiveTime)`
- 增强距离惩罚（×50），确保发射点靠近客户

### v3.1 (2026-02)

**无人机均衡分配**

- 无人机任务改为轮询分配
- 移除固定的无人机分配染色体段
- 解码时动态计算分配

### v3.0 (2026-02)

**发射点优化重构**

- 新发射点变异算法（历史方向 + 随机探索）
- 距离惩罚促使GA优化发射点位置
- 移除续航约束检查
